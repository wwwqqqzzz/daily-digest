## 🧊 冷知识

### 波利尼西亚人为何突然向东航行

1700 年间，波利尼西亚人一直在西太平洋的岛屿间航行，从未越过一条看不见的线。然后，大约公元 1000 年左右，他们突然掉头向东——发现了夏威夷、复活节岛，最终到达了南美西海岸。

有意思的是，**没有人确切知道为什么。** 考古学家确认了东行的证据——波利尼西亚人的语言和 DNA 遍布太平洋岛屿——但对于触发这次大迁徙的原因，主流学界至今在争论：是小冰河期改变了风向？是人口增长导致资源紧张？还是某种社会变革（比如宗教或酋长制度）激发了探险文化？

《海洋奇缘》（Moana）把莫阿娜塑造成一个叛逆少女出海拯救族人，但真实历史比电影更离奇也更模糊——一群没有文字、没有金属工具、靠星星和浪涌导航的航海者，在没有 GPS 甚至没有指南针的时代，穿过了地球上最广阔的水域。

### JustVibe: Find your wave

> [!tip] 简介
> 一个以情绪为核心的社交音乐发现平台，根据你当前的"vibe"匹配合适的音乐和同频的人。

### FetchSandbox

> [!abstract] 简介
> 一个 LLM 应用沙盒测试平台，让开发者在隔离环境里安全地测试 AI Agent 行为。

FetchSandbox 拿了 322 个 upvote，分数 63。这个品类在 2026 年属于刚需——随着 AI Agent 越来越多地接入文件系统、浏览器、Shell 等敏感接口，安全测试的诉求在快速上升。它的定位介于传统的 API sandbox（如 Postman 的 mock server）和 Agent 安全评估框架之间。

比较有意思的是：FetchSandbox 的卖点不是"能测什么"，而是"出了问题也能恢复"——每次测试自动生成快照，Agent 搞坏了环境可以一键回滚。对做 Agent 开发的团队来说，这个功能可能比覆盖率报告更值钱。

### Second Brain for AI v2

> [!note] 简介
> 一个自托管的记忆层——给 Claude、ChatGPT、Cursor 提供持久化上下文，让 AI 不再每次对话都"从零开始"。

279 个 upvote，分数 58。这款产品（以及 Vibe-Trading 同天上 GH Trending）体现了 2026 年中期的一个明确趋势：AI 的记忆和上下文正在从对话窗口走向外部化。Second Brain for AI v2 基于 Cloudflare Workers + D1 + Vectorize 构建，开源（MIT），部署两分钟搞定。

与同类产品对比：Mem.ai 是全托管但不开源，Rewind 聚焦本地数据抓取，Second Brain 走的是"你控制数据 + 所有 AI 工具通用"路线。它的 MCP 兼容性意味着不只是聊天机器人的记忆，而是 Cursor 的编码上下文、Claude 的项目记忆、ChatGPT 的对话历史都能共享同一个持久层。

### 《Starts With A Bang》播客第131期——重子生命周期

我有段时间没听 Ethan Siegel 的播客了，这期标题听起来像教科书章节——"重子生命周期"——但内容比名字有趣得多。大意是说重子（质子和中子这类构成可见宇宙大部分质量的粒子）不是永恒存在的：它们会衰变、重组，在恒星内部经历"生命周期"，最终变成光子或其他粒子散入星际空间。

我特别在意的一个点是：他说"你身体里的每一个质子都在等待死亡"。这个说法有争议（质子衰变至今未被实验证实），但拿来思考时间尺度很有意思——质子半衰期的下限在 10³⁴ 年左右，比宇宙年龄还大 24 个数量级。你本质上是用一群几乎永生的东西暂时组成了一个人形。

斯皮尔伯格又拍外星人了，这次叫《揭秘日》（Disclosure Day）。看标题还以为是《第三类接触》的续作——他说过那是他所有电影里最个人化的一部。但评论说这部电影"悬念充足、深意不足"——美国人拍外星人总在两极间晃动：要么是《降临》那种严肃语言学寓言，要么是《独立日》那种爆米花。斯皮尔伯格以前能在中间找到平衡（《ET》就是），但这次似乎偏向了神秘感堆砌。

说实话，没看正片没法判断评论是否公平。但有一点说得在理：如果一部外星人电影结尾只有悬念没有主题，那就只是个更贵的怪奇物语单集。

### 当你不再阅读纸质书时，你会失去什么

这个话题每隔一两年就会冒出来一次，但 Big Think 这篇写得有新鲜感——不是"纸质书手感更好"那种老生常谈，而是**专注力的肌肉记忆**。说纸质书阅读时大脑形成的是空间记忆：你记得一个情节大概在书的哪个位置、左边页还是右边页、靠近书脊还是靠近边缘。这种空间锚点是数字阅读（无限滚动 + 无固定位置）无法提供的。

我只用一个实验就验证了这个说法：回想上个月读的书，能想起纸质书封面和当时读的场景，但我记不起 Kindle 上读了什么。不是内容没意思，是大脑缺少了空间坐标去抓取记忆。

### 问伊桑：是什么决定了可观测宇宙的大小？

Ethan Siegel 的问答专栏又来了。这个问题看似基础——可观测宇宙大小由光速 × 宇宙年龄决定——但精彩在于"但是"。因为宇宙在膨胀，早期发出的光不是走过 138 亿光年来到我们这里的，光源已经退行到了更远的地方。所以可观测宇宙的半径大约是 465 亿光年，不是 138 亿。

最让人恍惚的是：**可观测宇宙的边界就是信息的边界。** 不是那里有什么墙，而是从更远处发出的光永远到不了我们。这个事实比任何科幻设定都更接近"宇宙在说：你只能知道这么多"。

## 📦 GitHub

### Dicklesworthstone/destructive_command_guard

AI Agent 的安全问题越来越紧迫了——这个 Rust 写的工具直接在命令执行前做一层"危险操作过滤"，拦截 git 和 shell 里那些 AI 可能误触的毁灭性命令（`git push --force`、`rm -rf /` 等）。不是什么新概念（Bash 有 `set -u`、Git 有 pre-commit hooks），但专门为 AI Agent 场景设计——因为 Agent 不会像人一样犹豫一下再确认。

### ColeMurray/background-agents

一个开源的"background agents"编码系统，TypeScript 写。单日增长不多（16），但方向有意思——不是让 AI Agent 和你对话式编程，而是让它在后台持续工作：分析代码库、修 bug、跑测试。和 Cursor Agent 模式的区别是：Agent 不打断你的工作流，异步产出结果。这可能代表了 AI 辅助编程的下一个阶段——从"你跟 AI 一起写"到"你写你的，AI 干 AI 的"。
