## 🧊 冷知识

### 金星的一天比一年长

在太阳系的所有行星中，金星拥有一个最反直觉的时间系统：它的自转周期（243个地球日）比公转周期（225个地球日）还要长。这意味着金星上的一天比一年还长。

更有意思的是，金星是太阳系中唯一一颗反向自转的大行星——如果你站在金星表面，太阳会从西边升起、东边落下。加上它极端的大气压力（地球的92倍），相当于在地球上900米深的海底漫步。直到1960年代雷达天文学证实之前，科学家们一直以为金星的自转跟地球差不多快。

第一个发现金星缓慢反向自转的线索来自1950年代的无线电观测，但数据太模糊没人敢确认。后来麦哲伦号探测器用雷达测绘才最终揭示：这颗距离地球最近的邻居，隐藏着太阳系中最奇异的节奏。

### Blaxel Agent Drive: A shared filesystem for AI agents

> [!tip] 简介
> 给 AI Agent 一个共享文件系统——不是另一个聊天界面，而是把内存、上下文和工具挂载到同一个虚拟磁盘上。

Blaxel 的思路跟之前那些 AI 集成产品不太一样。它不是在 UI 层面做拼接，而是在文件系统层面做：你的 Agent 能读写同一个命名空间里的文件，包括代码库、文档、笔记、数据表。所有连接的 Agent 共享一个实时同步的工作区。

跟传统的 Agent 框架（LangChain、AutoGPT）相比，Blaxel 解决了一个很实际的痛点：Agent 之间没有共享上下文。每个 Agent 启动时都是"失忆"的，需要反复注入相同的数据。Blaxel 让文件系统充当 Agent 的"长期记忆层".

当前竞品里，AgentGPT 和 CrewAI 走的是编排路线（orchestrating multiple agents），而 **Blaxel** 走的是基础设施路线（shared filesystem）。前者更适合多步骤任务链，后者更适合需要持续数据访问的场景。创始人提到接下来会加版本控制和访问权限管理，这对团队使用很关键。

### Caw: Open source web terminal multiplexer for AI agents

> [!example] 简介
> 给 AI Agent 一个浏览器里的终端多路复用器——有点像 tmux，但是跑在 Web 上，Agent 和人都能用同一个终端。

Caw 解决了一个很具体但很关键的工程问题：当你启动多个 Agent 任务时，它们的日志、错误输出、stdout 都混在一堆，没法追踪哪个 Agent 在做什么。Caw 在 Web 终端里实现了会话隔离、分屏、历史搜索和 Agent 专属输出通道。

Maker 在 PH 上提到它受 tmux 启发但做了几个关键改进：一是持久化（Agent 断开后 session 不掉），二是 JSON 友好的输出格式（方便 Agent 解析），三是 embeddable（可以直接嵌入到 Agent 的 Web UI 里）。

一个有意思的区别：传统终端复用器（tmux/screen）是为人类设计的，屏幕布局和快捷键都针对键盘交互。Caw 的布局逻辑默认是"Agent 优先"——Agent 的输出自动格式化到可解析的缓冲区，人只需要在必要时介入。

### Fable Flight: Learn to fly with a live AI instructor

> [!abstract] 简介
> 浏览器里的 AI 飞行学校——3D 驾驶舱、天气系统、实时交通、AI 教官坐在副驾，用最新的模型模拟真实飞行教学。

这个产品在今天的榜单上排第三。它不是用 AI 飞行模拟器来娱乐，而是真正的飞行入门训练：AI 教官会用自然语言解释仪表、指导降落、在紧急情况下给指令。多视角 3D 驾驶舱支持景观、天气和空中交通。

从 maker 的描述看，它使用了 GPT-5.6 级别模型来做实时语音教学。跟传统的微软模拟飞行 2024 或 X-Plane 相比，Fable Flight 最大的差异点是有个"真人"教官坐在旁边——不是预设的语音提示，而是根据你的操作动态响应。

当然，它不会给你 FAA 认证，但作为入门工具非常实用：在地面学理论花时间，在空中学操作花钱——Fable Flight 能帮你在模拟器里把肌肉记忆先建立起来。

### NotifyBridge: Dead-Simple IoT Push Notifications for Makers & Hobbyists

> [!info] 简介
> 不想给 IoT 设备写一整套后端？NotifyBridge 就是给 Maker 和硬件爱好者用的零配置推送服务。

自己动手做智能家居项目时，最烦的就是给设备配推送通知：你要写后端 API、管理 WebSocket 连接、处理设备离线重连、还要考虑安全问题。NotifyBridge 抽象了所有这些——提供一个简单的 API 端点和 SDK，几分钟就能让 ESP32/Raspberry Pi 给你的手机发推送。

跟 AWS IoT Core、Blynk 这类产品比，NotifyBridge 的定位更轻量：它不做设备影子、不做规则引擎、不做数据分析。就做一件事：**让设备能给你发消息**。Maker 文化里一直缺这样的产品——要么太重（AWS IoT），要么太封闭（Blynk 有生态锁）。

### Cubby Clipboard: Windows clipboard history that searches inside screenshots

> [!note] 简介
> Windows 终于有了一个真正好用的剪贴板管理器——而且能搜截图里的文字。

Windows 自带的 Win+V 在基础使用上还行，但它有个明显的短板：重启后历史就丢了，而且完全不支持对截图内容进行搜索。Cubby 用 Windows 自带的离线 OCR 引擎，让你的剪贴板历史不仅持久化，还在截图里的文字上建立了全文索引。

Maker 说它起源于 PastePaw 的 GPL fork，然后在可靠性、隐私和 Windows 原生体验上做了大量重写。所有数据本地加密存储，不依赖云服务。跟 Paste（macOS）和 Ditto（Windows）相比，Cubby 的差异化就是 OCR 搜索——Ditto 不支持，而 Cubby 把"我好像复制过一张图，里面有个电话号码"这个场景做到了一键搜索。

### 休假的价值不在于提高工作效率

有一个观点最近一直在脑子里转，今天读 Big Think 的新书摘编找到了呼应：关于休假的真正意义。

主流职场叙事从来都是把年假包装成"生产力管理工具"——好好休息才能更好地工作。但 Big Think 这本书摘选（来自新书 *Big Time Off*）提出了一个完全不同的角度：**休假的价值，根本不是让你回去后更有效率，而是让你在休假期间，重新体验"作为人"而不是"作为劳动力"的感觉。**

我特别在意的点是，它区分了两种时间：一种是"被工具化的时间"——也就是你度假时还在回邮件、调 Slack、想想下周的 deadline；另一种是"不服务于任何目的的时间"。后者才是休假真正稀缺的东西。大意是：现代人已经失去了"没事可做"的能力——哪怕躺在沙滩上，也会觉得应该做点什么"建设性"的事。

人择疲劳不只是在大脑层面。有一个说法让我印象深刻："当你休了一个真正的假——不看消息、不做计划、不思考下一步——你其实是在提前体验退休。"听起来有点抽象，但仔细想想，人一生被工作和目标驱动的阶段只占一小部分，后面大部分时间你不需要向任何人证明自己的效率。提早练习"不做什么"的能力，可能比提早准备养老金更重要。

读完我在想，最近几个月每次休假，脑子里那个"回去之后要做的事情清单"从来没有真正放下过。也许下一段假期前应该做一个实验：把手机通知关掉，提前在自动回复里写好"我不在，不看消息"，然后真的不看。试试看三天后是什么感觉。

[新书《Big Time Off》摘选，来自 bigthink.com](https://bigthink.com/books/big-time-off/)

### 19分钟读懂复仇心理学

19分钟的访谈，对一个话题的拆解密度令人意外。

普林斯顿的心理学家跟 Big Think 做了一期关于复仇心理学的深度对话。大意是：复仇不是人类进化中的"意外功能"，而是一种在特定条件下演化出来的合作稳定机制。关键不在于你受到伤害后想报复——这很正常——而在于复仇行为本身在群体中传递了一个信号："这个人惹了我，会付出代价。"

最让我惊讶的一点是：真正有效的复仇，不是暴力或对抗，而是**展示不计代价也要维护公平的决心**。这解释了为什么一些看似"非理性"的复仇行为反而在被观察到的群体中起到了威慑作用——因为正是因为"不理性"，才让潜在的挑衅者知道自己无法被轻易收买或吓退。

有一组对照实验：在博弈游戏里，愿意自损利益去惩罚不公平对手的玩家，在后续轮次中被故意亏待的概率显著更低。换句话说，"不计后果的报复者"反而获得了更多的合作机会。

文章也提到不是所有文化对复仇的评价都一样。在个人主义文化里它被看作失控，在集体主义文化里它有时被看作责任的履行。这让我想起之前读到的关于荣誉文化的一些内容——在美国南方，对轻微冒犯的强烈反应曾经是一种必需的生存策略。

对我而言，这19分钟的价值不在于学会"怎么报复"，而在于理解了一个微妙的事实：人有时候在意的不是损失本身，而是被当作可以随便对待的人。那口气咽不下去，不是因为"亏了"，而是因为"你凭什么觉得这样对我也可以"。

[The psychology of revenge, in 19 minutes — bigthink.com](https://bigthink.com/series/the-big-think-interview/psychology-of-revenge/)
